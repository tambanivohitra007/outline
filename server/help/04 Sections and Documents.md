Each condition is divided into **six sections**, and each section is backed by a full Outline document where you write the actual content.

## The six default sections

| Section | Purpose | What to write |
|---------|---------|---------------|
| **Risk Factors/Causes** | Identify what predisposes patients to this condition | Genetic factors, lifestyle risk factors, environmental causes, demographic patterns |
| **Relevant Physiology** | Explain the underlying biology | Pathophysiology, affected organ systems, disease mechanisms, biomarkers |
| **Complications** | Document potential complications | Comorbidities, progression risks, organ damage, quality-of-life impacts |
| **Solutions** | Therapeutic interventions organized by care domain | NEWSTART+ lifestyle interventions, protocols, dosing, practical recommendations |
| **Bible & Spirit of Prophecy** | Scripture and Spirit of Prophecy references | Relevant Bible passages, Ellen G. White quotations, theological context for health principles |
| **Ideas for Potential Research** | Document research gaps | Unanswered questions, proposed study designs, areas needing more evidence |

## Writing section content

Each section has a backing document. To edit it:

1. Open the condition in the condition editor
2. Find the section in the **Sections** tab
3. Click **Open Editor** to open the full Outline document editor

The document editor supports:

- Rich text formatting (headings, bold, italic, lists, tables)
- Images and file attachments
- Internal links to other documents
- Code blocks and math equations
- Collaborative real-time editing (when multiple users open the same document)

## Section document lifecycle

Section documents follow the condition\u2019s status:

| Condition status | Document state | Visible in collection? |
|-----------------|----------------|----------------------|
| Draft | Unpublished draft | No |
| Review | Published | Yes \u2014 team members can find and read them |
| Published | Published | Yes |

When you transition a condition from Draft to Review (via **Submit for Review**), all section documents are automatically published and added to the collection\u2019s document tree. When you move back to Draft, they are unpublished and removed from the collection tree.

## Creating a missing document

If a section does not have a backing document (this can happen if document creation failed), click the **Create Document** button in the section panel. The new document will be created as:

- A **draft** if the condition is in Draft status
- **Published** (and added to the collection) if the condition is in Review or Published status

## Previewing content

The section panel shows a **rendered preview** of the document content directly in the condition editor. This preview shows formatted text including headings, lists, tables, and links. You do not need to open the full editor to read the content.

## AI-generated content

Each section has an **AI Generate** button that produces a draft based on the condition name and section type. The generated content appears in a preview area where you can:

- **Copy & Edit** \u2014 copies the content to your clipboard and opens the document editor so you can paste and refine it
- **Dismiss** \u2014 closes the AI preview without using the content

AI-generated content is a starting point. Always review, verify, and edit the content before considering it complete.
