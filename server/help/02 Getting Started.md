This guide walks you through creating your first condition treatment guide, from start to finish.

## Step 1: Navigate to Conditions

Click **Conditions** in the sidebar to open the conditions list. This is where all treatment guides for your team are managed.

## Step 2: Create a new condition

Click the **New Condition** button. You will be asked for:

| Field | Required | Description |
|-------|----------|-------------|
| Name | Yes | The condition name (e.g., "Type 2 Diabetes Mellitus") |
| SNOMED Code | No | The SNOMED CT concept code for this condition. Use the autocomplete to search. |
| ICD Code | No | The ICD-10/ICD-11 code (e.g., "E11") |
| Collection | No | An existing collection to store documents in. If left empty, a dedicated collection is created automatically. |

When you create a condition, the system automatically:

1. Creates a **collection** to hold all the condition\u2019s documents
2. Creates **six sections**, each with a backing document:
   - Risk Factors/Causes
   - Relevant Physiology
   - Complications
   - Solutions
   - Bible & Spirit of Prophecy
   - Ideas for Potential Research
3. Sets the condition status to **Draft**

## Step 3: Write section content

Each section has a document where you write the actual content. Click **Open Editor** on any section to start writing in Outline\u2019s full-featured editor.

You can also use the **AI Generate** button on each section to get a draft starting point.

## Step 4: Add interventions

Switch to the **Interventions** tab. Link therapeutic interventions from the NEWSTART+ care domains. For each intervention you can set:

- The care domain it belongs to
- Evidence level (how strong the research support is)
- Recommendation level (how strongly you recommend it)
- Clinical notes

## Step 5: Add evidence

Switch to the **Evidence** tab. Search PubMed or ClinicalTrials.gov to import studies, or add entries manually. Each evidence entry can be linked to a specific condition or intervention.

## Step 6: Add Scripture references

Switch to the **Scripture** tab. Search for Bible verses or Spirit of Prophecy quotations and attach them to the condition.

## Step 7: Submit for review

When your draft is ready for team feedback, click **Submit for Review** in the condition header. This publishes all section documents into the collection so your team members can see and read them.

## Step 8: Publish

After review, click **Publish** to finalize the treatment guide. Published conditions are visible to all team members and can be exported.
